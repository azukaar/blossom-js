CONTEXT

The context is the most basic concept to use Blossom. It is your to-go way to store any volatile data, like request responses, arrays of users' objects, etc... Two things need to be considered when manipulating your context :

Does it need to be in the context ? If no, you will induce additonal pointless render cycles

Where ? this is crucial. Setting your ctx too deep in the DOM will cause issues, setting it too high will reduce performance. Basically most of the case the easy to solve this is :

  • if this is a user related data (like a todo list) it will be accessed from a lot of different places, therefore, set it on the default scope (the body). Or some place high
  • if this is directly correlated to a component, set it directly on this component : example, a request response, a processing result. (See : l-json)

In order to give maximum control on your scope, you will need to use . This component will allow you to set a context and limit its scope. this.ctx this.ctx

Note that in special cases you can sandbox the context with . Only use this if you know what you are doing.
