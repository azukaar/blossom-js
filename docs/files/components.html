BASIC COMPONENT CREATION

In Blossom-js, you define custom web element, like you would define a React-like component. In order to create a component you must create an object extending Blossom.Components.

class MyComponent extends Blossom.Component { render() { return 'hello world'; } }

Then, you can proceed to register your component to the global registry. To do so, you must provide the class, and a name for it. The name is used to asociate the component to the DOM. Exemple if the name is mysite-link, you will have to do to use it. The name must :

  • Use lower case characters
  • follow the pattern `namespace-name`

class MyComponent extends Blossom.Component { render() { return 'hello world'; } } Blossom.register({ name: 'mysite-link', element: MyComponent })

Here the list of available data. Remember that `this` is also the HTML element in the DOM itself, therefore any HTMLElement attributes/methods are available.

    this.ctx
    Access the context of the element. Warning: You should avoid reading/setting the context from a component unless that it is the point of it. Instead, read props and set local context via setCtx (local context means that the ctx will be set on the component itself instead of being set somewhere in the parent elements)
    this.props
    Access the props of the component. Those are basically HTML attributes on your component.

Here is a list of the methods implentable.

    render
    Called on render, return a string of the content
    setCtx
    Set the context on this particular component.
    onUpdate
    Called after render, use as post-hook
    onMount
    Called every time the element is attached to the DOM. Note that a re-render of the parent element will cause children to unmount / mount again.
    onUnmount
    Called every time the element is dettached from the DOM.
